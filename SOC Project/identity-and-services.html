Identity & Core Services

Phase Overview

This phase establishes the identity infrastructure and core network services that form the foundation of the enterprise environment. These components are critical for authentication, authorization, service discovery, and system communication.

From a security perspective, identity systems such as Active Directory represent a primary target for attackers, making their configuration and monitoring essential for both detection and defense.

Identity Architecture Design

  • Centralized Authentication: Active Directory domain (corp.local)
  • Name Resolution: DNS for internal service discovery
  • Dynamic Addressing: DHCP for endpoint configuration
  • Time Synchronization: NTP for log consistency and authentication

These services are tightly integrated and form the backbone of enterprise identity and network operations.

Active Directory Structure

Organizational Design

The Active Directory structure was designed to reflect a typical enterprise hierarchy, including separation of users, departments, servers, and workstations.

  • Organizational Units (OUs) for departments and infrastructure
  • User and group management aligned with business roles
  • Separation between servers, workstations, and domain controllers

Security relevance:
AD structure directly impacts access control, privilege management, and potential attack paths such as privilege escalation.
