Security Stack & Detection Architecture

Overview

The security stack is designed to provide full visibility across endpoint, network, and authentication activity within the environment. It enables detection of malicious behavior through log aggregation, correlation, and rule-based analysis.

The architecture combines multiple data sources to simulate a real SOC environment, where detections are not based on a single event, but on correlated activity across systems.

Security Components

  • Wazuh SIEM: Centralized log collection, correlation, and alerting engine
  • Sysmon: Deep endpoint visibility (processes, network connections, file activity)
  • Suricata IDS: Network-level threat detection
  • Windows Event Logs: Authentication and system-level events

Each component provides a different layer of visibility, and together they form a comprehensive detection pipeline.

Detection Pipeline & Data Flow

Logs are generated from multiple sources and processed through a centralized pipeline:

  • Endpoints generate telemetry (Sysmon, Windows Events)
  • Network traffic is analyzed by Suricata IDS
  • Wazuh agents collect and forward logs
  • Wazuh manager performs correlation and rule matching
  • Alerts are generated and mapped to MITRE ATT&CK techniques

This pipeline enables detection of complex attack scenarios by correlating multiple events, rather than relying on isolated indicators.

Detection Capabilities

  • Endpoint telemetry collection (process creation, network connections)
  • Authentication and system event logging (Windows Event Logs)
  • Network traffic monitoring and alerting (Suricata IDS)
  • Centralized log aggregation and normalization (Wazuh SIEM)
  • Custom rule support for event-based detection
