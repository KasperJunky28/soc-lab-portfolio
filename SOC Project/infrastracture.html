Infrastructure Setup

Phase Goal

Build a fully functional lab environment with proper network communication between all machines to simulate a real organizational network.

Steps

VM Deployment

Created multiple virtual machines to simulate different components of an enterprise environment.

Why this matters:
A controlled lab environment enables safe testing, attack simulation, and security analysis without affecting real systems.

  • Windows Server - Domain Controller, File Server
  • Windows 10 - Clients
  • Ubuntu - Web server, Wazuh SIEM server
  • Kali Linux - Attacker Machine
VM's preview

VM Network Configuration

Configured internal networking between all virtual machines, including separation between internal network and internet access (NAT).

Why this matters:
Understanding network architecture is critical for detecting suspicious traffic and identifying lateral movement within an organization.

  • Internal Network setup - Corp network/attacker network
  • NAT for internet access
