Infrastructure Design & Deployment

Phase Overview

This phase focuses on building the foundational infrastructure required to simulate a realistic enterprise environment. The goal is to create a stable, segmented, and fully functional lab that supports identity services, enterprise applications, and security monitoring.

The infrastructure is designed to enable controlled attack simulation while maintaining full visibility across systems, users, and network activity.

Design Principles

  • Isolation: Separation between internal infrastructure and attacker environment
  • Realism: Replication of enterprise components (AD, file server, endpoints)
  • Visibility: Ensure all systems generate logs for monitoring and detection
  • Scalability: Modular design allowing expansion of services and attack scenarios

Implementation Steps

Virtual Machine Deployment

Multiple virtual machines were deployed to represent different roles within an enterprise environment, including identity services, endpoints, servers, and attacker systems.

Key Components:

  • Windows Server – Domain Controller and File Server
  • Windows 10 – User endpoints (HR, IT)
  • Ubuntu Server – Web server and Wazuh SIEM
  • Kali Linux – Adversary simulation

Why this matters:
Using role-based virtual machines enables realistic simulation of enterprise operations, user behavior, and attack surfaces.

Network Configuration & Segmentation

Network connectivity was configured to simulate both internal corporate communication and controlled external attacker access.

Configuration Highlights:

  • Internal corporate network for domain-joined systems
  • Dedicated attacker network for adversary simulation
  • NAT configuration to enable controlled internet access
  • Dual-network interfaces on critical systems (e.g., SIEM)

Why this matters:
Proper segmentation is essential for modeling real-world attack paths, including initial access, lateral movement, and detection opportunities.
