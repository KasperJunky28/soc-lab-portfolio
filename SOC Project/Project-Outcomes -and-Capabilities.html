Project Outcomes & Capabilities

Project Overview

This project simulates a full enterprise security environment, combining infrastructure deployment, identity management, network configuration, attack simulation, detection engineering, and security hardening.

The environment was designed to replicate real-world security challenges, including misconfigurations, detection gaps, and attacker behavior across multiple stages of an attack lifecycle.

The project demonstrates the complete transition from a vulnerable system to a hardened, monitored, and detection-capable environment.

Attack → Defense Lifecycle

The project follows a full attack lifecycle, starting from external reconnaissance and progressing through internal compromise and data exfiltration, followed by defensive improvements and validation.

  • External Attacks → Nmap scanning, brute-force, HTTP flood
  • Initial Access → Credential compromise
  • Lateral Movement → SMB enumeration and access to internal shares
  • Data Exfiltration → Retrieval of sensitive files

Defensive Phase:

  • Security Hardening → Authentication, network, and access control improvements
  • Monitoring → Logging and SIEM integration
  • Detection → Custom rules and correlation logic
  • Validation → Full attack replay under hardened conditions

Infrastructure & System Engineering

Built and configured a complete enterprise-like lab environment from scratch, including servers, endpoints, and network architecture.

  • Deployed Active Directory Domain Controller
  • Configured DNS and DHCP services
  • Created organizational units, users, and group policies
  • Built and configured a File Server with multiple shared directories
  • Configured Windows and Linux machines across the network
  • Designed and implemented internal network segmentation

This required understanding of enterprise infrastructure design, system configuration, and network communication.

Network Security & Protection

  • Implemented Fail2Ban to prevent SSH brute-force attacks
  • Configured iptables rules to mitigate HTTP flood (DoS)
  • Analyzed attack traffic and validated blocking mechanisms

These controls introduced active network-level defense, reducing exposure to automated attacks and service disruption.

Identity & Access Management

  • Configured password policies using Group Policy
  • Implemented account lockout mechanisms
  • Hardened user accounts and removed insecure configurations

These controls significantly reduced the likelihood of credential compromise and unauthorized access to domain resources.

Logging, Visibility & Endpoint Monitoring

  • Enabled Windows auditing for object access (Event ID 4663)
  • Configured Sysmon for endpoint visibility
  • Collected and analyzed security logs

Visibility into system activity is essential for detecting malicious behavior and understanding attacker actions.

SIEM & Detection Engineering

  • Deployed and configured Wazuh SIEM
  • Integrated log collection from endpoints
  • Created custom detection rules
  • Implemented correlation logic between events

Example correlation: Login event (4624) + File access (4663) → Data exfiltration detection

This demonstrates the ability to move beyond raw logs into meaningful detection and behavioral analysis.

Security Concepts Demonstrated

  • Least Privilege
  • Defense in Depth
  • Prevention vs Detection
  • Visibility as a security requirement
  • Correlation-based detection

Final Analysis

This project demonstrates a complete security lifecycle: from building infrastructure, simulating attacks, identifying weaknesses, implementing defenses, and validating detection capabilities.

The environment evolved from a vulnerable system into a layered security model combining prevention, monitoring, and detection.

This reflects real-world security operations, where multiple controls must work together to protect systems effectively.

Security is not a single control, but a combination of layered defenses working together.
