Internal Attacks

Overview

Following successful initial access via SSH brute-force attack on the SIEM server, post-compromise activities were performed to simulate attacker behavior within the internal network.

  • Credential discovery
  • Lateral movement using SMB
  • Access to sensitive data
  • Data exfiltration

This phase demonstrates how an attacker can move from initial access to full data compromise.

Attack Chain

  • Initial Access (SSH)
  • Credential Discovery
  • SMB Enumeration
  • Lateral Movement
  • Data Access
  • Exfiltration

Credential Discovery (SIEM)

Description

After gaining access to the SIEM server, the attacker searched for sensitive information stored locally.

Execution

  • Command used: ls -la
  • Hidden file discovered: .backup.txt
  • Command used: cat .backup.txt

Command Breakdown

  • ls: Lists visible files in the current directory.
  • ls -la: Lists all files, including hidden files (starting with '.').

Evidence

Connecting to SIEM01 CLI using SSH

Hidden credentials file discovered

Credentials inside backup file

Execution Findings

  • Hidden file (.backup.txt) identified
  • Plaintext credentials successfully extracted

Result

  • Credentials discovered: ✔
  • Access to sensitive data: ✔

MITRE ATT&CK

  • T1552 – Unsecured Credentials

Analysis:
Plaintext credentials were discovered in a hidden file, representing poor credential management and insecure storage practices.

SMB Enumeration + Lateral Movement (SMB Access)

Description

Using the discovered credentials, the attacker enumerated SMB shares and attempted to access internal resources on the file server.

Execution

  • Command used: smbclient -L //file01 -U corp\\testuser
  • Command used: smbclient //file01/Management -U corp\\testuser
  • Attempted access to Finance and HR (Denied)
  • Successful access to Management share

Command Breakdown

  • smbclient: A tool used to interact with SMB (Windows file sharing).
  • -L: Lists all available shared resources on the target system.
  • //file01: Target file server.
  • -U corp\\testuser: Authenticates using domain credentials.
  • smbclient //file01/Management: Connects to a specific shared folder.

Execution Evidence

SMB shares enumeration

Access denied (Finance/HR)

Access granted (Management)

Execution Findings

  • Multiple shares identified (HR, Finance, Management)
  • Access denied to restricted shares (HR, Finance)
  • Access granted to Management share

Result

  • Shares enumerated: ✔
  • Lateral movement achieved: ✔
  • Unauthorized access to Management share: ✔

MITRE ATT&CK

  • T1021.002 – SMB
  • T1078 – Valid Accounts

Data Access & Exfiltration

Description

After successfully accessing the Management share, the attacker identified and retrieved sensitive data. The file was then transferred to an external system for further analysis, completing the data exfiltration phase.

Execution

  • Command used: ls
  • Command used: get management_confidential_report.pdf
  • Command used: scp transfer to attacker machine

Command Breakdown

  • ls: Lists files within the shared directory.
  • get management_confidential_report.pdf: Downloads the file from the remote SMB share.
  • scp: Transfers the file from the compromised system to the attacker machine.

Execution Evidence

Sensitive file identified and downloaded

File transferred to attacker machine

Extracted file content

Execution Findings

  • Sensitive management file identified
  • File successfully downloaded from SMB share
  • File transferred to external attacker machine

Result

  • Sensitive data accessed: ✔
  • Data exfiltration completed: ✔
  • Detected: ❌ (No file-level detection)

MITRE ATT&CK

  • T1039 – Data from Network Share
  • T1041 – Exfiltration Over Command Channel

Analysis:
The attacker successfully accessed and exfiltrated sensitive management-level data due to overly permissive access controls.

Detection & Visibility

Detection Evidence

SIEM detected SMB authentication

The SIEM detected successful authentication between the SIEM server and the file server.

Detection Gap

  • No detection of file access
  • No alert on sensitive file retrieval
  • No correlation of SMB activity to data exfiltration

Analysis:
While authentication activity was detected, no alerts were generated for file access or data exfiltration. This represents a critical visibility gap where attackers can access sensitive data without detection.

Attack Outcome & Detection Summary

Attack Result Summary

  • Initial access (SSH): ✔
  • Credential discovery: ✔
  • Lateral movement (SMB): ✔
  • Access to restricted shares: Partial (Management only)
  • Sensitive data accessed: ✔
  • Data exfiltration: ✔

Detection Summary

  • Authentication activity detected: ✔
  • SMB access detected: ✔
  • File access detected: ❌
  • Data exfiltration detected: ❌

Conclusion:
The attacker successfully compromised internal systems, accessed sensitive data, and exfiltrated it without triggering alerts for data access or data transfer activities.
