Post-Compromise Activity & Data Exfiltration

Phase Overview

Following successful initial access via SSH brute-force on the SIEM server, post-compromise activity was conducted to simulate attacker behavior within the internal network.

Key Objective:
Evaluate visibility and detection coverage for internal attacker activity after initial compromise.

Attack Chain

  • Initial Access (SSH)
  • Credential Discovery
  • SMB Enumeration
  • Lateral Movement
  • Data Access
  • Data Exfiltration

This sequence reflects a realistic attacker progression from foothold to full data compromise.

Credential Discovery – Local Enumeration

Description

After gaining access to the SIEM server, the attacker searched for sensitive information stored locally.

Execution

  • Command used: ls -la
  • Hidden file discovered: .backup.txt
  • Command used: cat .backup.txt

Command Breakdown

  • ls: Lists visible files in the current directory.
  • ls -la: Lists all files, including hidden files (starting with '.').

Security impact:
Storing credentials in plaintext exposes critical systems to compromise without requiring advanced techniques.

Evidence

Connecting to SIEM01 CLI using SSH

Hidden credentials file discovered

Credentials inside backup file

Execution Findings

  • Hidden file (.backup.txt) identified
  • Plaintext credentials successfully extracted

Result

  • Credentials discovered: ✔
  • Access to sensitive data: ✔

MITRE ATT&CK

  • T1552 – Unsecured Credentials

Analysis:
The discovery of credentials in a hidden file highlights weak credential management and lack of secure storage practices.

SMB Enumeration + Lateral Movement (SMB Access)

Description

Using the discovered credentials, the attacker enumerated SMB shares and attempted to access internal resources on the file server.

Execution

  • Command used: smbclient -L //file01 -U corp\\testuser
  • Command used: smbclient //file01/Management -U corp\\testuser
  • Attempted access to Finance and HR (Denied)
  • Successful access to Management share

Command Breakdown

  • smbclient: A tool used to interact with SMB (Windows file sharing).
  • -L: Lists all available shared resources on the target system.
  • //file01: Target file server.
  • -U corp\\testuser: Authenticates using domain credentials.
  • smbclient //file01/Management: Connects to a specific shared folder.

Security impact:
Valid credentials enable attackers to move laterally across systems and access internal resources.

Execution Evidence

SMB shares enumeration

Access denied (Finance/HR)

Access granted (Management)

Execution Findings

  • Multiple shares identified (HR, Finance, Management)
  • Access denied to restricted shares (HR, Finance)
  • Access granted to Management share

Result

  • Shares enumerated: ✔
  • Lateral movement achieved: ✔
  • Unauthorized access to Management share: ✔

MITRE ATT&CK

  • T1021.002 – SMB
  • T1078 – Valid Accounts

Analysis:
While access controls prevented entry to some sensitive shares, the attacker successfully accessed the Management share, demonstrating insufficient access restrictions.

Data Access & Exfiltration

Description

After successfully accessing the Management share, the attacker identified and retrieved sensitive data. The file was then transferred to an external system for further analysis, completing the data exfiltration phase.

Execution

  • Command used: ls
  • Command used: get management_confidential_report.pdf
  • Command used: scp transfer to attacker machine

Command Breakdown

  • ls: Lists files within the shared directory.
  • get management_confidential_report.pdf: Downloads the file from the remote SMB share.
  • scp: Transfers the file from the compromised system to the attacker machine.

Security impact:
Unauthorized access to sensitive business data represents a critical breach scenario with direct organizational impact.

Execution Evidence

Sensitive file identified and downloaded

File transferred to attacker machine

Extracted file content

Execution Findings

  • Sensitive management file identified
  • File successfully downloaded from SMB share
  • File transferred to external attacker machine

Result

  • Sensitive data accessed: ✔
  • Data exfiltration completed: ✔
  • Detected: ❌ (No file-level detection)

MITRE ATT&CK

  • T1039 – Data from Network Share
  • T1041 – Exfiltration Over Command Channel

Analysis:
The attacker successfully retrieved and exfiltrated sensitive data without triggering alerts, demonstrating a major visibility gap.

Detection & Visibility Analysis

Detection Evidence

SIEM detected SMB authentication

Observed Detection

  • Authentication events were logged and visible
  • SMB access activity was partially visible

Detection Gap

  • No monitoring of file access activity
  • No alerts for sensitive data retrieval
  • No correlation between authentication and data access

Analysis:
The environment provides visibility into authentication events, but lacks deeper monitoring of user activity and data access.

Attack Outcome & Detection Summary

Attack Result Summary

  • Initial compromise: ✔
  • Credential discovery: ✔
  • Lateral movement (SMB): ✔
  • Sensitive data accessed: ✔
  • Data exfiltration: ✔

Detection Coverage

  • Authentication visibility: ✔
  • Lateral movement visibility: Partial
  • File access monitoring: ❌
  • Data exfiltration detection: ❌

Conclusion:
The attacker successfully moved through the environment and exfiltrated sensitive data without detection at the data access level.

This demonstrates a critical gap in visibility and highlights the need for file auditing, behavioral correlation, and SIEM-based detection rules.
