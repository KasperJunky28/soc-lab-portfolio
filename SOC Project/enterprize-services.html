Enterprise Services & Data Exposure

Phase Overview

This phase introduces core enterprise services that store, process, and expose organizational data. These systems represent critical assets and are primary targets for attackers.

The goal is not only to deploy functional services, but to simulate realistic data access patterns, permission structures, and logging behavior that can later be used for detection and analysis.

File Server – Data Access & Permissions

Shared Folders & Access Control

A centralized file server was configured to simulate organizational data storage, including department-based access control and permission enforcement.

  • Created shared directories representing business units (HR, Finance, IT)
  • Assigned permissions based on user groups
  • Validated access behavior from domain-joined endpoints

Security relevance:
File servers are a primary target for attackers due to sensitive data exposure. Misconfigured permissions can lead to unauthorized access and data exfiltration.

Web Server – Internal Application Simulation

Service Deployment

An internal web server was deployed to simulate organizational applications and generate network and application-level activity.

  • Configured static IP using Netplan
  • Installed and enabled Apache2
  • Hosted internal web content
  • Validated connectivity from domain endpoints

Security relevance:
Web servers are common entry points for attackers and generate valuable logs for detecting suspicious activity such as scanning, exploitation attempts, and abnormal access patterns.
